Roadmap to Web Dev

A clean-slate plan: new email, new GitHub account, 60 small projects across six stages — HTML → CSS → JavaScript → Frontend → Backend → Fullstack. Every stage has 10 projects, and within each stage, project 10 is meaningfully harder than project 1.

Account & repo setup

Do this once, before commit 01. Five minutes now saves a messy profile later.

  • Create the gmail your-email@example.com and verify it.
  • Sign up at github.com with that email. Pick a username you'd be fine putting on a resume (e.g. your-github-username).
  • Add a short bio + a real (or neutral) avatar — empty profiles read as inactive.
  • Enable two-factor auth on the new account immediately.
  • Create one "meta" repo named roadmap or learning-log to track progress (you can even host this very page there as index.html via GitHub Pages).
  • Set a consistent commit identity: git config user.email "your-email@example.com"

repo naming

Use stage-number-project-name, e.g. 01-04-pricing-table. It sorts cleanly on your profile and signals progression to anyone browsing.

every repo needs

A short README (what it is, what you practiced, a live demo link via GitHub Pages/Netlify/Vercel if applicable), and a license is optional but tidy.

Rules to keep the account looking real

commit honestly

Small, frequent, descriptive commits over a long history beat one giant "final upload" commit. Build each project over 2–4 commits minimum.

deploy every frontend repo

GitHub Pages, Netlify, or Vercel — a live link in the README turns a code dump into a portfolio piece.

write the README first

Before coding, write what the project does and why. It clarifies scope and gives you a finish line.

don't skip the boring ones

The early "easy" projects in each stage exist to build muscle memory — rushing past them shows up as shaky fundamentals later.

Suggested terminal flow for kicking off any new project:

# inside your projects folder
mkdir 03-06-weather-app && cd 03-06-weather-app
git init
git branch -M main
echo "# Weather App" >> README.md
git add . && git commit -m "init: scaffold project"
gh repo create your-github-username/03-06-weather-app --public --source=. --push
