Express routingREST conventions (verbs + status codes)An ORM/driver (Prisma or Mongoose)Schema designEnvironment variablesTesting endpoints with Postman/curl

How to approach it

  1. Get a database connection working and confirm it with one hardcoded query before writing routes.
  2. Build routes one at a time — GET all → GET one → POST → PUT → DELETE — testing each in Postman before the next.

How to approach it

Four habits that make the difference

  • commit honestly
    Three to four real commits per build beats one "final upload." A commit history is a story — let it read like one.
  • deploy everything
    Every frontend repo gets a live link in the README. A code dump is not a portfolio piece; a working link is.
  • README first
    Write what the thing does and why before you write a line of code. It's a finish line, not paperwork.
  • don't rush the easy ones
    The early builds in each leg exist to build muscle memory. Skipping them shows up as shaky fundamentals two months later.
